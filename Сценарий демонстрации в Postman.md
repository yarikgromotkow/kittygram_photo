# Сценарий демонстрации API в Postman

**Проект:** Kittygram Cats API  
**Преподаватель видит:** как работает JWT, защита объектов, валидации, фильтрация  
**Время демо:** ~10 минут

---

## Подготовка (сделай заранее)

1. Запусти проект:  
   ```
   python manage.py runserver 8001
   ```  
   или через Docker:  
   ```
   docker-compose up -d
   ```

2. Открой Postman → Import → выбери `Kittygram_Cats_API.postman_collection.json`

3. Убедись, что в разделе **Variables** коллекции стоит:  
   - `base_url` = `http://127.0.0.1:8001`  
   - (если Docker) `base_url` = `http://localhost`

4. Тестовые данные уже загружены через `load_test_data.py` — это несколько кошек и пользователей.

---

## Шаг 1 — Показываем публичный доступ (Гость)

> **Что объяснять:** «Просмотр списка кошек не требует авторизации — это соответствует роли Гостя»

**Запрос 2.1 — GET /api/cats/**
- Отправить без токена
- Показать: приходит 200, список кошек с полями id, name, color, birth_year, age, owner, achievements

**Запрос 2.2 — GET /api/cats/?color=Gray**
- Показать фильтрацию: приходят только серые кошки
- Упомянуть: также работают `?owner=`, `?search=`, `?ordering=name`

**Запрос 2.3 — GET /api/cats/?search=** (любое имя из тестовых данных)
- Показать поиск по имени

---

## Шаг 2 — Авторизация (JWT)

> **Что объяснять:** «Для изменения данных нужен JWT-токен. Его получают через Djoser»

**Запрос 1.1 — POST /api/auth/jwt/create/**
```json
{
  "username": "testuser",
  "password": "testpass123"
}
```
- Показать: приходит 200 с полями `access` и `refresh`
- Скопировать значение `access` в переменную `token` коллекции (или вставить вручную в Authorization → Bearer)

---

## Шаг 3 — Создание кошки (авторизованный пользователь)

> **Что объяснять:** «Создать кошку может только авторизованный пользователь. Владелец подставляется автоматически из токена»

**Запрос 2.4 — POST /api/cats/** (с Bearer токеном)
```json
{
  "name": "Барсик",
  "color": "Black",
  "birth_year": 2022,
  "achievements": [
    {"achievement_name": "Чемпион"}
  ]
}
```
- Показать: приходит 201, в ответе поле `owner` = id текущего пользователя, поле `age` вычислено автоматически
- Запомнить `id` созданной кошки (допустим, 5)

---

## Шаг 4 — Обновление своей кошки

> **Что объяснять:** «PATCH позволяет обновить отдельные поля. Работает только для владельца»

**Запрос 2.6 — PATCH /api/cats/5/** (с Bearer токеном)
```json
{
  "birth_year": 2021
}
```
- Показать: 200, поле birth_year обновлено, age пересчитался

## Шаг 5 — Негативные тесты (самая интересная часть)

> **Что объяснять:** «Теперь покажем три типа ошибок, которые API обрабатывает корректно»

**Тест 5.1 — 400 Bad Request (валидация)**  
Запрос: POST /api/cats/ с `"birth_year": 1800`  
→ Показать: 400, `{"birth_year": ["Проверьте год рождения!"]}`  
**Вывод:** CatSerializer.validate_birth_year отклоняет нереалистичный год

**Тест 5.2 — 401 Unauthorized**  
Запрос: POST /api/cats/ БЕЗ заголовка Authorization  
→ Показать: 401, `"Authentication credentials were not provided."`  
**Вывод:** IsAuthenticatedOrReadOnly блокирует неавторизованных

**Тест 5.3 — 403 Forbidden**  
- В Postman вставить токен другого пользователя (или удалить токен и поставить чужой)  
- Запрос: PATCH /api/cats/{id чужой кошки}/  
→ Показать: 403, `"You do not have permission to perform this action."`  
**Вывод:** IsOwnerOrReadOnly — защита на уровне объекта

## Шаг 6 — Swagger (бонус)

> **Что объяснять:** «API полностью задокументирован автоматически через drf-yasg»

- Открыть в браузере: http://127.0.0.1:8001/swagger/
- Показать: список всех эндпоинтов, нажать на любой, показать Try it out
- Авторизоваться через Authorize → добавить `Bearer <токен>`
- Отправить GET /api/cats/ прямо из Swagger

## Что говорить в ответ на вопросы

| Вопрос | Ответ |
|--------|-------|
| Почему JWT, а не Session? | JWT stateless — сервер не хранит сессии, подходит для API |
| Что такое Djoser? | Библиотека-надстройка над DRF для аутентификации: регистрация, логин, токены |
| Как работает IsOwnerOrReadOnly? | Переопределяет has_object_permission: SAFE_METHODS разрешены всем, остальное — только obj.owner == request.user |
| Зачем AchievementCat? | Явная промежуточная модель для M2M позволяет расширить таблицу дополнительными полями в будущем |
| Зачем Nginx? | Reverse proxy + раздача статики без Django, разгружает Gunicorn |
| Как запустить без Docker? | python manage.py runserver 8001 после pip install -r requirements.txt и migrate |
